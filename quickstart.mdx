## Instalação

Para utilizar a biblioteca `js-plugin-lib` em um novo projeto de plugin, você deve instalá-la diretamente do repositório Git.

<Steps>
  <Step title="Inicie um novo projeto Node">
    Crie uma pasta para seu plugin e inicialize o NPM.
    ```bash
    mkdir meu-novo-plugin
    cd meu-novo-plugin
    npm init -y
    ```
  </Step>
  <Step title="Instale o SDK">
    Execute o comando de instalação utilizando SSH. Garanta que suas chaves SSH do GitHub estão configuradas.
    ```bash
    ssh -T git@github.com
    npm install git+ssh://git@github.com:L-0-W/RevelePortal-Plugin.git
    ```
    
    > **Nota:** Como o projeto é construído em TypeScript, ao instalar do Git, o NPM executará automaticamente o script `prepare` para transpilar o código (criando a pasta `dist/`).
  </Step>
  <Step title="Crie o arquivo principal">
    Crie seu arquivo `index.js` (ou `.ts` se preferir) e instancie o Plugin.
    ```javascript index.js
    const { Plugin } = require("js-plugin-lib");

    // O nome deve ser IDÊNTICO ao nome definido no arquivo plugin.toml
    const plugin = new Plugin("meu-plugin");

    plugin.on("connected", () => {
        console.log("Uhul! Estamos conectados ao orquestrador.");
    });
    ```
  </Step>
</Steps>

## O que acontece por baixo dos panos?

Quando a classe `Plugin` é instanciada:

1. O Node.js **pausa** e aguarda o orquestrador Rust enviar o comando `READY` via `stdin/stdout`.
2. Após o Handshake, um Socket IPC (Unix Domain Socket ou Named Pipe) é estabelecido.
3. O evento `connected` é emitido, indicando que é seguro interagir com o banco de dados e UI do Revele.
